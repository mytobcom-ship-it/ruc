MapMatchSvr 알고리즘 플로우차트

MapMatchSvr/bin/config.ini 및 소스(RawLogFetcher/Worker, MapMatch, BeginMapMatch, ContinueMapMatch, GISUtil::SgmtMatch)를 근거로 재구성한 실제 제어흐름입니다. 순차 나열이 아니라 분기(◇)·루프·피드백을 포함한 개발용 flowchart입니다. (2026-07-22 기준 소스)

MATCHED / 성공 SKIP / 저신뢰 ERROR / 완전실패 / 배제 중간 처리 단계

3단 backpressure: 정상(<400) 500ms 고정폴링 · 준혼잡(400~800) 2~10초 선형증가 · 과부하(≥800) 10초로 사실상 정지. CServer::LogMonitorStatus()(30초 주기)는 이 상태를 로그로만 남기고 제어에는 관여하지 않음.

depth 확장은 조기확정(early-exit) 최적화입니다: 최적후보가 링크 끝점에 스냅됐거나(IsBoundaryClamped) 방위각이 매우 나쁠 때(IsPoorAngleFit, 방향비용≥14.99)만, depth 여유가 있으면 진출 링크까지 넓혀 비교합니다. maxstep은 공백구간 이동거리가 크면 MM_STEP_EXTEND_DIST(50m)당 +1, 최대 +3까지 가산됩니다. MAP_MATCH_FAIL은 오직 "후보 링크 자체가 하나도 없을 때"만 발생합니다.
